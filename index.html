动态规划股票问题总结

发表于 2023-12-26 分类于 算法 本文字数： 3.2k 阅读时长 ≈ 6 分钟

股票问题总结

  • 最多买卖一次

    • 笔者认为本题最好使用贪心来求解

      • class Solution{
    public int maxProfit(int[] prices){
        int min = Integer.MAX_VALUE;
        int res = 0;
        for(int i = 0; i < prices.length; i++){
            min = Math.min(min, prices[i]);
            res = Math.max(res, prices[i] - min);
        }
        return res;
    }
}

      • 因为可以买卖无限次，那么疯狂交易就好了，碰上利润为负的话不要交易就可以了

  • 最多买卖两次

    • 无法贪心了，老老实实用动态规划

    • 买卖两次比买卖多次要复杂一些

    • 我们将一天分为五种状态

      • 0：没有操作
      • 1：第一次买入
      • 2：第一次卖出
      • 3：第二次买入
      • 4：第二次卖出

    • dp[i][0-4]表示第i天的最大利润是多少

    • 初始化

      • dp[0][0] = 0
      • dp[0][1] = -prices[0]
      • dp[0][2] = 0
      • dp[0][3] = -prices[0]
      • dp[0][4] = 0

    • 递推公式

      • dp[i][0]= dp[i - 1][0]
      • dp[i][1] = Math.max(dp[i - 1][1], dp[i- 1][0] - prices[i])
      • dp[i][2] = Math.max(dp[i - 1][2], dp[i - 1][1] + prices[i])
      • dp[i][3] = Math.max(dp[i - 1][3], dp[i - 1][2] - prices[i])
      • dp[i][4] = Math.max(dp[i - 1][4], dp[i - 1][3] + prices[i])

    • 其实优化一下比较好

      • 主要优化空间以及状态0其实可以不管
      • dp[0] 表示第一次买入：dp[0] = -prices[i]
      • dp[1]表示第一次卖出：dp[1] = Math.max(dp[1], dp[0] + prices[i])
      • dp[2]表示第二次买入：dp[2] = Math.max(dp[2], dp[1] - prices[i])
      • dp[3]表示第二次卖出：dp[3] = Math.max(dp[3], dp[2] + prices[i])
    • Class Solution{
    public int maxProfit(int[] prices){
        int[] dp = new int[4];
        dp[0] = -prices[0];
        dp[2] = -prices[0];
        for(int i = 0; i < prices.length; i++){
            dp[0] = -prices[i];
            dp[1] = Math.max(dp[1], dp[0] + prices[i]);
            dp[2] = Math.max(dp[2], dp[1] - prices[i]);
            dp[3] = Math.max(dp[3], dp[2] + prices[i]);
        }
        return dp[3];
    }
}

  • 最多买卖多次，含冷冻期

    • 看卡哥的思路有些没明白状态1和状态2是怎么区分出来的

    • 所以看的是另外一位老哥的思路，发现跟简单一些，主要是在对冷冻期的处理上，只需要加上冷冻期前一天的最大利润就好了

    • 直接上代码

    • class Solution{
    public int maxProfit(int[] prices){
        if(prices == null || prices.length < 2){
            return 0;
        }
        
        int[][] dp = new int[prices.length][2];
        dp[0][0] = -prices[0]; //买入
        dp[0][1] = 0; //卖出
        //还要初始化第一天，因为dp[i - 2][1] - prices[i]
        dp[1][0] = Math.max(dp[0][0], - prices[1]);
        dp[1][1] = Math.max(dp[0][1], dp[0][0] + prices[1]);
        
        for(int i = 2; i < prices.length; i++){
            dp[i][0] = Math.max(dp[i - 1][0], dp[i - 2][1] - prices[i]); //买入，冷冻期前一天dp[i-2]
            dp[i][1] = Math.max(dp[i - 1][1], dp[i - 1][0] + prices[i]);
        }    
        return dp[prices.length - 1][1];

    }
}

    • 代码思路：主要就是两个状态：0表示买入，1表示卖出; 买入的递推公式就需要处理冷冻期的情况，只有两种情况，要么你前一天就是买入的状态，要么你就是当天买入的，注意是在当天买入的前提下，如果取得是dp[i - 2][1] - prices[i],证明i-1天你才刚卖出，第i天你就是冷冻期，这时候不能买入，这就和我们的前提冲突了，所以只能是i-2天卖出的状态

  • 最多买卖多次，含手续费

    • 含手续费可太容易了，三刷的时候一次过
    • 注意是买卖多次，只要把每次卖出的时候在扣掉手续费就可以了
    • 给出递推公式：
      • dp[i][0] = Math.max(dp[i - 1][0], dp[i - 1][1] - prices[i]) //买入
      • dp[i][1] = Math,max(dp[i - 1][1], dp[i - 1][0] + prices[i] - fee)//卖出

OSI 七层参考模型

  • 口诀：从下至上依次为“物、数、网、传、会、表、应”
  • 物理层：主要定义的是物理设备的标准。比如要定义网线的接口类型、光纤的接口类型，传输介质间的传输速率等。这一层的数据叫做比特，主要传输比特流，比特流就是由0或1转化为电流强弱来进行传输。
  • 数据链路层：建立逻辑连接、硬件地址寻址，差错校验等功能。这一层的数据叫做帧，数据在这一层会以帧为单位进行传输。在这一层封装MAC地址。
  • 网络层：进行逻辑地址寻址。**IP协议**就在这层
  • 传输层：TCP协议和UDP协议都在这层，这一层的数据叫做段
  • 会话层：通过传输层（端⼜号：传输端⼜与接收端⼜）建⽴数据传输的通路。主要在你的系统之间发起会话或者 接受会话请求。
  • 表⽰层：数据的表⽰、安全、压缩。主要是进⾏对接收的数据进⾏解释、加密与解密、压缩与解压缩等（也就是 把计算机能够识别的东西转换成⼈能够能识别的东西（如图⽚、声⾳等）。
  • 应⽤层：⽹络服务与最终⽤户的⼀个接⼜。这⼀层为⽤户的应⽤程序（例如电⼦邮件、⽂件传输和终端仿真）提 供⽹络服务。
